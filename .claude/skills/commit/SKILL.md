# Commit

## Instructions

Prepare a commit message for current changes. NEVER execute git write commands (git add, git commit, git push).

### Steps

1. Run `git status` to see all changes
2. Run `git diff` and `git diff --cached` to understand changes
3. Run `git log --oneline -10` to see recent commit style
4. Draft a commit message following the repository's conventions
5. Present the suggested commit message and list of files to include
6. STOP — do not run `git add`, `git commit`, or `git push`
